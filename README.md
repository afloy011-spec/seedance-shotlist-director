# seedance-shotlist-director

Claude-скилл: превращает сценарий (текст, синопсис, сцену) в режиссёрский шотлист для Seedance 2.0 — один самодостаточный HTML-файл с готовыми к копированию промптами и продакшн-трекингом.

## Как работает

1. На вход — сценарий любой длины и, опционально, кастомный style prefix.
2. Скилл разбивает его на сцены и 15-секундные промпты, каждый в строгой структуре: Style CORE + Lighting → Characters → Scene → CUT 1..N → ENDS ON → SFX.
3. На выходе — `shotlist.html`: открывается в браузере, кнопка Copy отдаёт промпт целиком для вставки в Seedance.
4. Файл прогоняется через `scripts/validate.mjs` — блокирующие критерии проверяются автоматически до показа пользователю.

## Структура скилла

```
SKILL.md                      ядро: режиссёрский метод, структура промпта, воркфлоу (~220 строк)
references/
  board-spec.md               механика HTML-борда: блоки, localStorage-контракт, схема Project Bible
  html-template.html          готовый скелет (CSS/JS копируются дословно, заполняются плейсхолдеры)
  worked-examples.md          эталонный промпт + эталонная ревизия
  extras.md                   9:16 / 1:1 варианты и монтаж под BPM (грузится только по запросу)
scripts/
  validate.mjs                zero-dep валидатор сгенерированного shotlist.html
tests/
  golden-scenarios.md         три стресс-сценария + чеклист критериев
examples/                     один сценарий, собранный оригинальным и текущим скиллом
```

Ядро грузится в контекст при каждом вызове; reference-файлы — только на том шаге, где нужны (worked-examples перед написанием промптов, board-spec + template перед сборкой HTML, extras — по триггеру). CSS/JS борда не генерируются заново, а копируются из шаблона — меньше поверхность для ошибок.

## Формат промпта

- **Style prefix** разделён на неизменное ядро (оптика, кожа, физика, continuity, 24fps) и строку Lighting, которая проектируется отдельно под каждую сцену (время суток, источники, мосты между сценами).
- **@ассеты** — персонажи, локации и пропсы ссылаются по именам (`@hero`, `@hero_wet`, `@bottle`), совпадающим с Elements в Higgsfield; варианты состояния (мокрый/сухой/в крови) — отдельные ассеты.
- **ENDS ON** — каждый промпт заканчивается описанием точного финального кадра; следующий промпт открывается с него. Между сценами проектируются match-cut'ы.
- **Диалоги** — реплики пишутся в катах с режиссурой подачи, аудио — diegetic only.
- Промпты всегда на английском (требование Seedance), описания сцен и UI — на языке пользователя.

## Продакшн-борд (HTML)

- **Asset Checklist** — первый блок: что собрать в Higgsfield до генерации (фейс-клоузап + фулбоди, локации в ¾, продакт-шиты, layout map для сложной геометрии).
- **Разметка рисков** — каждый промпт помечен цветным текстовым бейджем safe / tricky / high-risk с причиной (толпа, хореография, текст в кадре) и оценкой числа попыток; рекомендация — генерировать высокорисковые первыми.
- **Бюджет** — сводка сверху: целевой хронометраж, число промптов, секунды генерации, оценка кипер-секунд на промпт (`15s gen → ~3s final`).
- **Трекинг** — на каждый промпт: статус (not started / generating / retry / keeper), таймкод кипера, заметки. Хранится в localStorage с неймспейсом по слагу проекта — два борда не пересекаются, состояние переживает перегенерацию файла.
- **Repair Guide** — таблица «симптом → правка» для типовых фейлов генерации (дрейф лица, левитация пропсов, каша в хореографии, несклеивающиеся клипы).
- **Project Bible** — JSON-блок внутри HTML с фиксированной схемой: персонажи, ассеты, свет по сценам, хэндоффы, статусы. Новая сессия Claude восстанавливает контекст проекта из самого файла; ревизии не требуют пересказа.
- Файл без внешних зависимостей, работает офлайн; clipboard с фолбэком; экранирование HTML в промптах.

## Валидатор

```
node scripts/validate.mjs shotlist.html
```

Zero-dependency (Node ≥18). Проверяет ~47 пунктов: парсинг и обязательные ключи Project Bible, соответствие Bible ↔ DOM (сцены, id промптов), неймспейсинг localStorage, полноту каждого промпта (CORE, Lighting, Characters, Scene, CUT 1, ENDS ON, SFX), английский язык промптов, @ссылки, экранирование, контролы статус/кипер/заметки, риск-бейджи, зеркала перевода, отсутствие эмодзи и внешних ресурсов. Exit 1 при любом FAIL; дубли Lighting между сценами — warning. Скилл обязан прогнать валидатор после каждой генерации и ревизии.

## Дополнительно

- **Аспекты**: все промпты компонуются center-safe (ключевое действие в центральных ~40% ширины кадра); по запросу борд перекомпоновывается под 9:16 / 1:1 — планы теснее, блокинг в глубину, варианты `1a-v` / `1a-sq`.
- **Монтаж под музыку**: при заданном BPM каты нарезаются по тактам (240/BPM сек), действия расписываются по нумерованным битам; сетка сохраняется в Project Bible.
- **Ревизии**: правки применяются к тому же файлу — нумерация сцен и слаг стабильны, перегенерируется только затронутое; хэндоффы пересшиваются.
- **Границы**: скилл заточен под Seedance 2.0; для других моделей (Veo, Kling, Runway и т.д.) сохраняется режиссёрский метод, но контейнер промпта адаптируется, а не подставляется молча.

## Тесты

[tests/golden-scenarios.md](tests/golden-scenarios.md) — три стресс-сценария (минимальный вход; 8 сцен с машиной состояний персонажа; диалоговая драма со сменой ритма), 10 блокирующих критериев, 8 балльных, проба ревизии. Механическая половина критериев автоматизирована в `scripts/validate.mjs` (помечена `[auto]`); драматургия и семантика continuity проверяются вручную. Правки SKILL.md прогоняются по ним на регрессию.

## Установка (Claude Code)

Положить папку скилла целиком (SKILL.md + references/ + scripts/) в папку скиллов:

```
~/.claude/skills/seedance-shotlist-director/
```

Вызов: «сделай шотлист» / «make a shotlist» + сценарий.
